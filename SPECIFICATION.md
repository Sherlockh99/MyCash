# MyCash — Технічне завдання

## 1. Призначення системи

**MyCash** — мобільний додаток для ведення особистих фінансів на Android: облік доходів, витрат, переказів між рахунками, планування бюджету та звітність.

---

## 2. Функціональні вимоги

### 2.1. Рахунки (Accounts)

| Параметр | Опис |
|----------|------|
| Кількість | Декілька рахунків |
| Валюта | Гривня (UAH) — єдина валюта |
| Створення | Тільки вручну |
| Початковий баланс | Опціонально, можна створити «з нуля» |

**Поля рахунку:**
- Назва
- Тип (карта, готівка, рахунок — довільне введення)
- Початковий баланс (за замовчуванням 0)
- Дата створення

**Операції:** створення, редагування, видалення.

---

### 2.2. Категорії (Categories)

| Параметр | Опис |
|----------|------|
| Структура | Ієрархія: категорія → підкатегорія (2 рівні) |
| Налаштування | Повністю налаштовується користувачем |
| Типи | Окремі набори для доходів та витрат |

**Поля категорії (рівень 1):**
- Назва

**Поля підкатегорії (рівень 2):**
- Назва
- Батьківська категорія

**Операції:** створення, редагування, видалення категорій та підкатегорій.

---

### 2.3. Операції (Transactions)

**Типи операцій:**

1. **Дохід** — джерело (підкатегорія), рахунок, сума, дата, коментар
2. **Витрата** — підкатегорія, рахунок, сума, дата, коментар
3. **Переказ** — рахунок списання, рахунок зарахуння, сума, дата

**Загальні поля:**
- Дата й час
- Коментар (опціонально)

---

### 2.4. Періодичні операції (Recurring Transactions)

| Параметр | Опис |
|----------|------|
| Інтервал | Щомісяця |
| Створення | При першому запуску додатку в новий день |

**Логіка:** при першому відкритті додатку в новий календарний день перевіряються всі періодичні операції; якщо минула чергова планова дата — створюється звичайна операція за відповідним шаблоном.

**Поля шаблону періодичної операції:**
- Тип (дохід/витрата)
- Рахунок
- Підкатегорія
- Сума
- День місяця (1–31)
- Остання дата створеної операції (для розрахунку наступної)

---

### 2.5. Планування бюджету (Budget)

| Параметр | Опис |
|----------|------|
| Рівень | Тільки підкатегорії витрат |
| Ліміти | Окремий ліміт на кожну підкатегорію |
| Період | Місяць |

**Поля бюджету:**
- Підкатегорія витрат
- Ліміт (сума на місяць)
- Місяць/рік (або автоматично за поточний місяць)

**Звітність:** звіт план/факт — порівняння запланованого та фактичного споживання бюджету.

---

### 2.6. Резервні копії (Backup)

| Параметр | Опис |
|----------|------|
| Формат | JSON |
| Експорт | Ручний (кнопка) + автоматичний |
| Імпорт | Відновлення з файлу |

**Автоматичний бекап:**
- Момент: перший запуск додатку в новий день
- Після успішного створення: показати повідомлення про створення бекапу
- Якщо не створився: спроба повторити при наступному запуску

**Операції:** експорт (ручний + авт.), імпорт (вибір файлу).

---

### 2.7. Звіти та аналітика

| Параметр | Опис |
|----------|------|
| Період | Місяць |
| Відображення | Кругова діаграма витрат по категоріях; порівняння план/факт |

**Зміст звітів:**
- Кругова діаграма: витрати по підкатегоріях (або категоріях) за місяць
- Таблиця/список план vs факт по підкатегоріях

---

### 2.8. Налаштування (Settings)

**Мова:**
- Українська (за замовчуванням)
- Російська
- Англійська

**Тема:**
- Світла
- Темна

## 3. Архитектура

### 3.1. Стек технологій

| Компонент | Технологія |
|-----------|------------|
| UI | Jetpack Compose, Material3 |
| Навігація | Navigation Compose |
| Архітектура | MVVM |
| База даних | Room (SQLite) |
| Асинхронність | Kotlin Coroutines, Flow |
| DI | Hilt (опціонально) |

### 3.2. Структура пакетів

```
com.sh.mycash/
├── data/
│   ├── local/
│   │   ├── entity/       # Room entities
│   │   ├── dao/          # DAO інтерфейси
│   │   └── AppDatabase.kt
│   ├── repository/       # Репозиторії
│   ├── model/            # Domain-моделі
│   └── backup/           # Експорт/імпорт JSON
├── domain/
│   └── usecase/          # Use cases (періодичні оп., бекап, тощо)
├── ui/
│   ├── theme/
│   ├── components/       # Спільні UI-компоненти
│   ├── screens/
│   │   ├── dashboard/    # Головний екран
│   │   ├── accounts/     # Рахунки
│   │   ├── transactions/# Операції
│   │   ├── categories/   # Категорії
│   │   ├── planning/     # Бюджет
│   │   ├── reports/      # Звіти
│   │   └── settings/     # Налаштування
│   └── navigation/
├── di/
└── util/                 # Утиліти (локалізація, дати)
```

### 3.3. Модель даних (Room entities)

```
Account
  id: Long
  name: String
  type: String
  initialBalance: Double
  createdAt: Long

Category (рівень 1)
  id: Long
  name: String
  type: CategoryType (INCOME | EXPENSE)

Subcategory (рівень 2)
  id: Long
  categoryId: Long
  name: String

Transaction
  id: Long
  type: TransactionType (INCOME | EXPENSE | TRANSFER)
  amount: Double
  accountId: Long
  targetAccountId: Long?     // для TRANSFER
  subcategoryId: Long?       // для INCOME/EXPENSE
  date: Long
  comment: String?

RecurringTransaction
  id: Long
  type: TransactionType (INCOME | EXPENSE)
  amount: Double
  accountId: Long
  subcategoryId: Long
  dayOfMonth: Int            // 1-31
  lastCreatedDate: Long?     // дата останньої створеної операції

Budget
  id: Long
  subcategoryId: Long
  amount: Double
  month: Int                 // 1-12
  year: Int
```

### 3.4. Екрани та навігація

**Нижня навігація (tabs):**
1. **Головна** — сумарний баланс, останні операції, швидкі дії
2. **Операції** — список з фільтрами, додавання/редагування
3. **Звіти** — діаграми, план/факт
4. **Налаштування** — мова, тема, бекап

**Додаткові екрани (перехід з головної або меню):**
- Рахунки — список, CRUD
- Категорії — дерево категорій/підкатегорій, CRUD
- Планування — бюджет по підкатегоріях, CRUD

### 3.5. Локалізація

- `values/strings.xml` — українська (default)
- `values-ru/strings.xml` — російська
- `values-en/strings.xml` — англійська

Мова зберігається в SharedPreferences/DataStore.

### 3.6. Формат JSON бекапу

```json
{
  "version": 1,
  "exportedAt": "2025-03-10T12:00:00",
  "accounts": [...],
  "categories": [...],
  "subcategories": [...],
  "transactions": [...],
  "recurringTransactions": [...],
  "budgets": [...]
}
```
